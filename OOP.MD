## 객체 지향 프로그래밍에 대한 내용

객체 지향 프로그래밍을 이해하기 위해선 객체 지향 프로그래밍이라는 용어가 나오게 된 배경을 알아야 합니다.

객체 지향 이전의 프로그래밍 패러다임은 크게 두 가지로 나눌 수 있으며 다음과  같습니다.


1. __비구조적 프로그래밍__   
1. __절차적 프로그래밍__

그럼 비구조적 프로그래밍과 절차적 프로그래밍에 대해서 간단하게 알아보자면

__비구조적 프로그래밍__ 은 말 그대로 코드를 구조화하지 않고, 그냥 첫 줄부터 마지막 줄까지 차례대로 실행되는 형태를 말합니다. 그리고 코드의 흐름을 이동하는 goto문 같은 것을 사용하는 특징이 있습니다.

```c
// 예시 코드
int main(){
    
    int a = 0;
    label: // goto문으로 인해 a가 5보다 작은 경우 해당 위치로 이동함
    a += 1;
    if (a < 5){
        goto label; // label에 위치로 이동
    }
    return 0;
}
``` 
예시 코드를 보게되면 솔직히 읽기도 힘들고 , 빠르게 이해하는 것도 힘들다는 것을 알 수 있습니다. 그렇다면 비구조적 프로그래밍은 잘못된 프로그래밍 패러다임이라고 볼 수 있을까요?
  
그건 아니라고 당연하게 말할 수 있습니다. 프로그래밍 패러다임에는 옳고 그름이 있는 것이 아니고 , 단순히 방식이 다른 것입니다. 
어떤 경우에서는 어떤 프로그래밍 패러다임이 효율적인지는 말할 수 있으나,
다른 프로그래밍 패러다임을 썼다고 해서 그게 나쁜 것이 아니라는 것은 알고 계셨으면 합니다.

---
__절차적 프로그래밍__ 은 이제 위에서 알아본 __비구조적 프로그래밍과 다르게 소스코드를 여러 부분으로 나눠서 활용하는 프로그래밍 패러다임이며, '프로시저'를 이용하여 구조화 시키는 방식을 말합니다. 

여기서 '프로시저'는 \0

절차적 프로그래밍에서는 코드의 구조를 모듈화할 수 있는데, 모듈화를 하게되면 동일한 기능을 수행하는 코드를 재작성 하지 않아도 재사용 가능하며,라이브러리처럼 누군가가 만들어 놓은 기능을 사용하며 프로그램을 더 쉽게 개발 가능하다는 장점이 있으며, 또한 구조화된 코드는 다른 사람이 쉽게 읽기 가능하다는 장점도 있습니다.
